# Retrieval-System
Retrieval-система по статьям с arXiv

В проекте реализована двухэтапная retrieval-система для поиска научных статей по текстовому запросу. 

Система сочетает:

1. **Semantic search с bi-encoder**
2. **Re-ranking с cross-encoder**

Такой подход позволяет сначала быстро найти кандидатов по эмбеддингам, а затем более точно ранжировать результаты.

---

# Данные 

Используется корпус **arXiv metadata**, содержащий:

- `title`
- `abstract`
- `id статьи`

Текст документа формируется как title + abstract

Дополнительно используется **тестовый набор запросов**, где каждому запросу соответствует релевантная статья.

---

# Этап 1. EDA

Проведен анализ корпуса статей:

- объединены `title` и `abstract`
- проведена очистка текста
- исследована длина документов

Это позволяет выбрать оптимальный размер входной последовательности для модели.

---

# Этап 2. Реализация retrieval-системы

Система состоит из двух этапов:

## 1. Semantic search (bi-encoder)

На первом этапе используется **SentenceTransformer** для получения эмбеддингов (all-mpnet-base-v2).


Алгоритм:

1. кодирование всех документов корпуса
2. кодирование пользовательского запроса
3. поиск ближайших документов по косинусному сходству

Результат: top-K кандидатов.

В системе используется: top_k = 50.

## 2. Re-ranking (cross-encoder)

Для повышения качества используется **cross-encoder reranker**.

Он принимает (query, document) и предсказывает релевантность. Затем кандидаты переупорядочиваются по score модели.

Преимущество:

- bi-encoder — быстрый поиск
- cross-encoder — высокая точность ранжирования

---

## Реализация системы

Реализован класс ArxivRerankerSystem. Он включает:

- загрузку моделей
- кодирование документов
- поиск кандидатов
- reranking
- сбор статистики времени выполнения

Система также сохраняет:

- время кодирования
- время поиска
- время reranking

---

# Этап 3. Оценка качества системы

Для оценки используется метрика MRR@5 (Mean Reciprocal Rank). Она показывает, насколько высоко в выдаче находится правильный документ.

Итоговая метрика системы: MRR@5 = 0.9363, т.е. релевантный документ почти всегда находится в первых результатх поиска.

Данный результат существенно превосходит простой векторный поиск без reranking.

# Этап 4. Производительность

![Графики производительности каждой из частей системы](https://github.com/Uchiha6Sasuke/Retrieval-System/blob/main/time.png)

На основе полученных графиков производительности можно выделить следующие характеристики системы:
1. query encoding (bi-encoder):
    - среднее время 11.6 мс, работает быстро и предсказуемо;
2. vector search (util.semantic_search):
    - среднее время 1.5 мс, самая быстрая часть системы. даже полный перебор через матричное умножение на данном объеме данных работает почти мгновенно. внедрение сложных индексов на этих данных не даст заметного прироста общей производительности;
3. re-ranking (cross-encoder):
    - среднее время 132.9 мс, это узкое место системы. на этот этап приходится более 90% времени обработки запроса. разброс времени обусловлен разной длиной текстов кандидатов, которые обрабатывает модель.
